## 必须修改项（Agent 必须严格按以下方式修改）

当前网络主体（STFT → U-Net → Single Mask → Residual）已经合理，**不得修改整体架构**。仅允许修改以下三处，以保证与 SVMD 理论一致。

---

# ① 每个递归步骤维护独立中心频率（必须修改）

## 当前问题

目前 Criterion 中仅维护一个全局中心频率：

```python
self.register_buffer("omega", torch.tensor(0.0))
```

所有 step 共用同一个 ω。

这会导致：

* step0 更新 ω
* step1 覆盖 ω
* step2 再覆盖 ω

最终 J1、J2 始终围绕最后一次更新的 ω 工作，不符合 SVMD 中每个模态拥有独立中心频率的设计。

---

## 必须修改

改为维护：

```python
self.register_buffer(
    "omegas",
    torch.zeros(max_steps)
)
```

forward 中：

```python
omega = self.omegas[step]
```

更新时：

```python
self.omegas[step] = EMA(...)
```

不要再维护单个全局 omega。

---

# ② History Loss 恢复论文 Eq.(7)（必须修改）

## 当前问题

目前：

```python
loss += (hist_power * curr_power).mean()
```

只是频谱重叠惩罚。

虽然能够训练，但已经不是 SVMD 原论文中的 J3。

---

## 必须修改

History 中保存：

```python
{
    "mode_spec": ...,
    "omega": ...
}
```

对于 history 中每个模态：

先根据

```python
omega_i
```

重新构造论文中的 β 滤波器

```python
beta_i
```

然后按照论文 Eq.(7)

```text
J3 = Σ || β_i · û_new ||²
```

计算历史正交损失。

不要直接计算：

```python
hist_power * curr_power
```

---

# ③ Step Embedding 与 History 保持同步（建议按此实现）

当前：

```python
b = b + step_emb
```

已经能够区分不同递归步骤。

保持这种实现即可。

另外 History 保存时，同时保存对应 step 的中心频率：

```python
history.append({

    "mode_spec": mode_spec.detach(),

    "omega": criterion.omegas[step].detach()

})
```

以后 J3 直接读取：

```python
history[i]["omega"]
```

无需重新计算。
