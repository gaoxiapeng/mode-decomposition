# Neural SVMD 项目状态总结与问题清单

> 本文档汇总 Neural SVMD 从初始版本到当前的完整演进、当前模型状态、
> 以及真实风电信号测试暴露的核心问题与解决思路。
> 细粒度的逐函数改动记录见 `改动清单.md`；本文档是高层状态总结。

日期：2026-06-28

---

## 一、项目本质（一句话）

频域 U-Net（`NeuralSVMD`）+ 单掩码 + 残差递归，做**无监督**变分模态分解：
每步从当前残差里用一个掩码 M 抠出一个模态（mode=M·X，residual=(1−M)·X），
残差喂下一步，直到停止。损失为频域 SVMD 三项 `L = w1·J1 + w2·J2 + w3·J3`，
无重构损失（mode+residual=input 结构性成立）。

详细架构见 `CLAUDE.md`。

## 二、改动演进（四个阶段）

### 第一轮：按《修改方案.md》对齐论文
- 全局标量 `omega` → 按 step 的 `omegas[max_steps]` 向量。
- `history_loss`（J3）从频谱重叠惩罚恢复为论文 Eq.(7) 的 β 滤波器形式。
- 同步 `train_vmd.py` 的 step 调用；修了 periodic-save 未定义 `path`、resume 兼容两个 bug。

### 第二轮：修「高频提不出 + 后段混叠」（据 epoch30 日志诊断）
- **根因**：`omegas[step]` 是「跨样本+跨 epoch 平均质心」，锁死在低频
  `[~100 ~330 ~560 ~780 ~980]`，30 epoch 不动 → 高频模态拿不到参考频率 → 提不出。
- **改动**：J1/J2/J3 的 ω_L 改用「**当前模态逐样本实时质心**」（detached），
  `omegas` buffer 降级为只供日志。J2/J3 权重大幅抬高（w2 1→20，w3 2→40，alpha 50→10）。
- **效果**：omega 解锁覆盖到高频，三项损失同量级，loss 健康下降。

### 第三轮：修「单模态多峰合并」（据 vmd_3 测试图诊断）
- **根因**：第二轮 alpha 砍到 10（J1 弱 5 倍）+ w3=40 过强，模型学会「把剩余频率
  打包成一坨当新模态」的偷懒解——J1 罚不动、J3 又只看「别和历史撞车」。
- **改动**：alpha 10→50、w1 20→40、w3 40→20（w2=20 不动）。
  最终默认 `L = 40·J1 + 20·J2 + 20·J3`，alpha=50。
- **效果**：前段模态恢复单峰。

### 第四轮：推理停止判据 + 残差可视化
- **根因**：单掩码模型每步被迫输出 M，真实模态提完后仍反复抓同一残差热点 →
  产出大量中心频率相同的伪模态（撞 20 步上限）。
- **改动**：`test_vmd.py` / `demo_vmd.py` 的 `decompose` 加三重停止判据：
  ① 能量 < epsilon ② 残差能量平台（降幅 < 2%）③ 新模态中心频率与**任一**历史模态
  接近（< 50Hz，比相邻判定更稳，避免误杀非单调跳动的频率序列）。
- 模态图末尾新增一行**残差**（黑色，时域+频域），便于看「还剩多少没分干净」。

## 四、当前核心问题：真实风电信号无法正确分解

测试数据：`exp/wind/group_4_r10r30.xlsx` 第 4 列 `P_HESS`（风电/储能功率信号），
**每分钟一个采样点**（fs ≈ 1/60 ≈ 0.0167 Hz），约 380 点（≈6.3 小时）。
脚本：`Code/test_wind.py`，输出 `exp/wind/wind_decomp.png`。

### 现象
三个提取模态的中心频率全挤在 600~760Hz（标注频率，基于错误的 8kHz 假设），
频谱形状几乎一模一样 = **严重混叠**；真实信号中高频能量几乎全留在残差里
（残差能量 0.196，很大），完全没分解干净。

### 三层根因（按重要性）

1. **【头号】训练/测试分布严重不匹配——这是本质问题，改采样率救不了。**
   - 模型在 8kHz、模态散布 100~3900Hz 的**合成窄带正弦**信号上训练，学到的是
     **绝对频率带**的习惯（freq_grid 通道 + step embedding 都按绝对 Hz 学）。
   - `--sample_rate` 参数**只影响显示和停止判据，不进模型 forward**：模型永远按
     「512 点 FFT 的 bin 索引」处理，所以无论采样率标多少，它都往训练时熟悉的
     低频 bin 区去提 → 反复提同一带 = 混叠。
   - 真实风电功率是**宽带、非平稳、含强趋势**的信号（频谱是连续鼓包，非离散尖峰），
     模型从未见过这种分布。

2. **采样率标注错误（可修但不解决混叠）。**
   - 真实 fs ≈ 0.0167 Hz，奈奎斯特仅 0.0083 Hz，信号所有成分对应周期 ≥ 2 分钟。
   - 当前图的「4000Hz 横轴」假，差约 48 万倍。修 `--sample_rate 0.0167` 只能改对
     频率轴标注，不改变模型行为。

3. **信号太短（≈380 点 < n_fft=512）。**
   STFT 几乎没有有效时间帧，频率/时间分辨率都极差，是模态频谱长得一样的部分原因。

## 五点五、关键认知：模型按「归一化频率」工作，不按 Hz

> 这一节纠正一个常见误解，直接决定「采样率怎么改」「训练长度怎么选」。

### 核心事实
`models.py` 里 `freq_grid = torch.fft.rfftfreq(n_fft)` 是**归一化频率 [0, 0.5]**
（单位：cycles/sample，即「每个采样点转过多少个周期」），**没有乘 sample_rate**。
`losses.py` 算中心频率 ω 也用这条归一化轴。

因此 **`--sample_rate` 不进模型 forward、不进损失**，它只在 `compute_center_freq`
和画图时把归一化频率换算成 Hz **显示**。

**推论：**
- 改 `--sample_rate`：对模型分解行为**无任何影响**（改了也不变），只为把图上 Hz 标注正确。
- 真正决定模型能否分对的是：**信号长度（点数）** 和 **信号的归一化频率分布**是否与训练时一致。

### 采样率（仅为显示正确）
- 1min/点：`fs = 1/60 ≈ 0.0167` Hz → `--sample_rate 0.0167`
- 15min/点：`fs = 1/900 ≈ 0.00111` Hz → `--sample_rate 0.00111`

（Hz 数值很小不直观，风电更适合用「周期(分钟)」思考，见下表。）

### 归一化频率 ↔ 周期换算（核心）
```
归一化频率 f_norm = 1 / 周期(采样点数) = 采样间隔(min) / 周期(min)
```

**1min 采样**下各周期对应的归一化频率：

| 周期 | 2min | 10min | 30min | 1h | 2h | 6h | 12h | 24h |
|------|------|-------|-------|----|----|----|-----|-----|
| f_norm | 0.5 | 0.1 | 0.033 | 0.0167 | 0.0083 | 0.0028 | 0.0014 | 0.0007 |

**当前模型训练分布**：合成模态 f_norm ≈ 0.0125~0.49（即周期 **2min ~ 80min**，按 1min 采样）。

### 训练 / 测试信号长度选取规则
1. **信号长度（点数）≥ 关心的最长周期 × 5~10 倍**，否则该周期看不全：
   - 关心到 1h 周期(60 点) → 信号 ≥ 300~600 点
   - 关心到 6h 周期(360 点) → 信号 ≥ 2000~4000 点
   - 关心到日周期(1440 点) → 信号 ≥ 7000~14000 点（数天数据）
2. **n_fft ≤ 信号长度**，且 n_fft 决定低频分辨率（最低非零 bin 周期 = n_fft 点）：
   - n_fft=512 → 最低可分辨周期 512 点（1min 采样下 ≈ 8.5h）
   - 要看日周期 → n_fft 需加大到 1024~2048

### 两种情况的处置
- **A) 风电关心分钟~1小时级波动（周期 < 80min）**：归一化频率范围**已匹配**当前模型。
  主要工作 = 测试信号**取够长**（≥1024 点，别再是 380 点 < n_fft），用对的 `--sample_rate`
  显示。**可能不必重训**。
- **B) 关心多小时~日级周期（周期 > 80min）**：模型没见过这么低的归一化频率，**必须重训**：
  - 改 `generate_synthetic_signal`，让合成模态包含低频（长周期）；
  - `--signal_length` 加大到能覆盖数个目标周期；
  - `--n_fft` 提到 1024~2048。

### 还需用户确认
1. 风电信号真正关心的**周期范围**？（分钟级波动 / 小时级 / 日级）→ 决定走 A 还是 B。
2. **数据总量**？（仅 380 点，还是有数天/数周连续序列）→ 决定能否取够长窗口或重训。

## 五、解决思路（四条路，待决策）

> 这是「用合成数据训练的模型去推理分布完全不同的真实信号」的固有 gap，
> 不是 bug，也不是调参能解决的。四条路按代价/彻底程度排列：

### 路线 A：先修采样率看一版（最快，仅诊断用）
```bash
cd Code && python test_wind.py --sample_rate 0.0167 --out ../exp/wind/wind_g4_fs.png
```
- 优点：立刻看到正确频率轴和真实混叠程度。
- 缺点：**治不了混叠**，只是把问题看清楚。

### 路线 B：改用经典 VMD/SVMD 算法（最实用，推荐优先验证）
- 放弃神经网络，用经典变分模态分解（手写 ADMM 或 `vmdpy` 库）直接分解这段真实信号。
- 优点：不依赖训练分布，对短的非平稳信号更稳；学界处理风电信号普遍用经典 VMD。
- 缺点：需要人工指定模态数 K、惩罚因子 α 等参数；放弃了神经网络的端到端优势。

### 路线 C：重采样 + 当归一化频率喂现模型（折中尝试）
- 把信号重采样/补长到模型习惯的长度（如 16000 点），频率按 [0, 0.5] 归一化解释。
- 优点：复用现有模型，改动小。
- 缺点：仍是勉强迁移，分布不匹配的本质没解决，**很可能仍混叠**。

### 路线 D：用真实风电数据重训/微调模型（最彻底，最费时）
- 用真实风电信号（或类似的非平稳宽带信号）重新训练或微调，让训练分布匹配测试分布。
- 优点：从根上解决分布不匹配。
- 缺点：需要足够的真实数据、重新设计训练流程、时间成本高；
  无监督损失在真实信号上是否仍有效需要验证。
